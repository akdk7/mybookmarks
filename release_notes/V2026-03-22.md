Summary
This release introduces a full API integration stack and a much more capable prompt workflow editor, while simplifying link setup and improving large-board responsiveness. It also hardens API runtime behavior, security-sensitive paths, and keeps the wiki aligned with the new workflows.

Changes

Features

* Added a new API Integrations manager for connections, endpoints, response mappings, placeholder sources, actions, and built-in test flows.
* Added OAuth2 PKCE, Device Code, and Client Credentials support with vault-backed secret handling, token refresh, and safer request previews.
* Added Prompt Flow V1 with execution, API load, branch, and assign steps, workflow variables, conditional step visibility, and API-aware editor shortcuts.
* Added a simplified default link setup flow with Quick Add, flow presets, deep links into API settings, and richer placeholder details directly in the editor.

Improvements

* Improved response mapping with array, object, and expression modes, inline preview, path picking, clearer mode guidance, action-aware testing, and clickable usage navigation.
* Improved UI performance on large boards by moving hot editing paths off deep reactive state, narrowing cache invalidation, reducing render work, and softening expensive Windows Chromium effects.
* Refined dialog and prompt UX with responsive step controls, better dark-mode and form styling, consistent placeholder tables, and clearer backup activity summaries.

Fixes

* Fixed duplicate API calls in prompt and visual refresh flows, making GET and HEAD request reuse and cache behavior more reliable.
* Fixed several API and runtime edge cases, including vault unlock reload behavior, delayed link-title refreshes, placeholder source matching, stale visual refresh paths, and response-mapping preview issues.
* Hardened security-sensitive paths with sanitizing for group HTML, safer redirect handling, extra drag-and-drop permission guards, smarter groupId recovery, and explicit save failure reporting.

Documentation / Wiki Updates

* Added and updated wiki pages for API integration quick start, placeholder sources and actions, request bodies and overrides, response mapping recipes, and the main API dialog.
* Updated documentation for the placeholder prompt workflow, placeholder rules, backup activity log, sidebar feature overview, and the local test REST server.

Technical / Internal

* Expanded Playwright coverage for API integrations, prompt workflows, placeholders, performance, persistence, and security regressions.
* Added JMESPath and JSONata support libraries to power the new expression and assignment features.
